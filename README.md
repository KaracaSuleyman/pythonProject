## ðŸš€ Git & GitHub Workflow

### GitHub DOs
* **Branching:** Create a new branch immediately after being added as a collaborator.
* **File Staging:** Do not use `git add .` blindly. Specify the file path.
   * *Example:* `git add decorators.py`
   * (Tip: Use "Copy Path" > "Path from Content Root" in your IDE).
* **Updates:** Run `git pull` on the **master** branch before starting any new work.
* **Merging:** Always pull the latest changes on **master**, then switch to your branch and **merge**.
* **Integrity Check:** Regularly check that you are on your feature branch and NOT on **master**.
* **Commits:** Follow the format `git commit -m "date name"`.

### GitHub DON'Ts
* **Master Branch:** Strictly no direct development on the **master** branch.
* **Communication:** Do not push updates without communicating with the team.
* **Leadership:** Consult the **Lead** before making changes to shared or core modules.
